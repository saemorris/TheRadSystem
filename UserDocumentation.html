Installation Guideline

Unzip the thinger into the root directory of your php server.

$> tar -xvzf radsystem.tar.gz

From SQLPlus, run the setup.sql file.

$> sqlplus

Enter user-name: ...

Enter password: ...

SQL> @setup

SQL> exit;

Edit the ‘_database.php’ file to have the required oracle credentials.

$__username = "my_rad_username";

$__password = "my_rad_password";

An admin account has been setup for you with:

username: admin

password: admin

It is recommended that you change the password to this account.

User Manual

Login Module

To log in to the system, the user must enter their username and password and press Login. If an incorrect username or password is entered an error message will occur prompting the user to try again. This is will navigate the user to the home page where the user will have the option to logout by clicking on the “Logout” option or click “My Account” to view or edit their account information. After the have navigated to their account page, the user can choose “Change Password” where they will be required to provide their old password before changing to their new password, or they can choose to “Edit Account Info” which will allow the user to change any of their existing information providing they can enter the corresponding account password.

User Management Module

To add or modify user accounts, the user must first log in under an admin account. From the home page, press the “Manage Users” from the menu at the top. The user may then enter a query to search for users, or add a new account using the “Add Account” button. When searching users, the query will search by username, first name, last name, address, email, or phone. The query may contain up to 5 terms. If any of these fields matches, the user’s information is displayed. While not case sensitive, the details must be an exact match to return a record. Clicking the user’s ID allows their information to be edited. This includes the account type, username, password, profile information, and associated doctors. When adding users, only the username and personal details are displayed for editing. After creation, the user search page is displayed with the new user’s username automatically in the query if further edits need to be made.

IMPORTANT: If a patient is not assigned a family doctor by the admin user, they will not be shown in searches by users.

Report Generating Module

To generate a report, the user must first log in using an admin account. After a successful login, the home page is displayed where the user can select the option “Generate Report” and be taken to a page where the can specify the report they are looking for. The user enter values for a diagnosis and a year and clicks Submit, which displays all the results that match both of the given inputs. If the user does not enter both a diagnosis and a year an error message occurs detailing which value was missing.

Uploading Module

To upload a radiology record into the database, the radiologist must first login. Once logged in they will have the option to “Upload Record”, which will take the user to page where they can fill out the fields pertaining to a radiology record - the patient id, doctor id, radiologist id, test type, prescribing date, test date, diagnosis and description - and click “Upload”. This then takes the user to a page where they can upload a jpg image file and insert it into the record they just created and click “Upload”. Upon success the user is displayed the uploaded image as well as some stats on the image.  A ‘Home’ link is displayed at the top of this screen to return the user to the home page.

Search Module

To search the database the user must first log in to the database. Any user can search the database however the amount of information a user can view is restricted depending on their security level - a patient’s account can only view his own records, a doctor’s account can only view the records of their patients, a radiologist’s account can only view records in which they took the X-Rays, and an admin account can view all the records. Once the home page is displayed, the user can choose to search the database with keywords or by date range. Any keywords the user enters is matched against existing records for test type, first name, last name, diagnosis, and test description and all results with the user’s security clearance are returned. To search by date range the user must enter both dates in the form YYYY/MM/DD, which will match the date with both the prescription date and the test date, returning all results within the user’s security clearance level.

Data Analysis Module

To view a data breakdown, the user must first log in using an admin account. After a successful login, the home page is displayed where the user can select the option “Data Analysis” and be taken to a page where the can specify the report they are looking for. The user may specify any combination of patient name (first name and last name), a test type and a time period. If the user does not want to specify any of these values they can leave them as the default “All”.  When the user wants to view the number of images per month they must first provide the year who’s months they wish to view, and likewise when they wish to see the breakdown by week they must provide the year and months who’s weeks they want to examine. The weeks are divided by the days of the month, so depending on the month some Week 5’s will have less days than others. By clicking Submit a table(s) is generated with a summary of the images taken.
